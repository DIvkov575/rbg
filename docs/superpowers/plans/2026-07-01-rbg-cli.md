# rbg Scriptable CLI — Engine-Backed Verbs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give rbg a scriptable, engine-backed command layer and route the real `rbg` binary's non-interactive verbs through it — `ls`, `create`, `run`, `send`, `read`, `kill`, `adopt` — replacing the `internal/client`-backed implementations. This makes the completed Engine the thing the CLI drives, and retires the orphaned `internal/localagent` package.

**Architecture:** Layer 4 (front-end) of `docs/HLD-rbg-clean-architecture.md` (§4.6 CLI parity, §5.1 cmd→engine). The `core`+`host`+`engine` layers are complete and merged. This slice adds `internal/cli` — a thin dispatcher that maps parsed args to Engine calls and renders results — plus the `cmd/rbg` wiring to build a real Engine and dispatch through it. The CLI depends on a small `Ops` interface (satisfied by `*engine.Engine`), so it is fully unit-testable with a fake — no real SSH/claude. Rendering the inventory reuses the existing pure lenses in `core`.

**Scope boundary (important — read before scoping):** The interactive **dashboard** (`cmd/rbg/dash.go` → `internal/tui`, with its own `Deps` shape over `internal/client`/`internal/queue`) is the Phase-4 rewrite and is OUT of scope here. Consequently `internal/client`, `internal/queue`, and `internal/tui` must remain (the dashboard still uses them); they are retired when the dashboard is rewritten, not now. This slice retires only `internal/localagent`, which is already orphaned (verified: no non-test importers). The `deploy`, `attach`, `ping`, and `dash` verbs keep their current implementations.

**Tech Stack:** Go 1.26 (module `github.com/divkov575/rbg`), stdlib only. Reuses `internal/engine`, `internal/core`, `internal/config`, `internal/run`, and `internal/render` (existing transcript renderer). No new dependencies.

**Scope of this plan (from HLD §2):**
- **F9** scriptable surface: every observe/stage/control operation available as a non-interactive verb.
- Surfaces **F1/F2** (`create`+`run`), **F3** (`ls`, `adopt`), **F4** (`send`, `kill`), **F5** (sync-first via `run`), **F8** (`read`) through the binary.
- **Not in scope:** the dashboard rewrite (Phase 4) and the retirement of `client`/`queue`/`tui` it enables; `deploy`/`attach`/`ping`/`dash` verbs (unchanged); a `CreateAndRun` convenience (a `run --new` could come later); typed degradation errors (noted forward).

**Verified facts (grounded 2026-07-01):**
- `engine.New(cfg *config.Config, r run.Runner, storePath, home string) (*engine.Engine, error)`; methods `List() ([]core.Agent, error)`, `Create(core.Agent) (core.Agent, error)`, `Run(name string) error`, `Send(name, task string) error`, `Read(name string) ([]byte, error)`, `Kill(name string) error`, `Adopt(name string) error` (all merged & tested).
- `core.Agent` fields: Name/Repo/Dir/Task/Session/Where(`Local`/`Remote`)/State(`Held`/`Running`/`Done`)/Origin(`Managed`/`Foreign`)/Sync/RunAt/Pid.
- `core.OnMachine`, `core.GroupByRepo` lenses exist (for future views; `ls` here is a flat render).
- `internal/render.Stream(lines []string, out io.Writer)` renders a transcript's JSONL lines to human text (used by the old `read`).
- `internal/localagent` has NO non-test importers (verified via grep) — safe to delete.
- Current `cmd/rbg/main.go` `parse()` returns an `*inv{verb,name,task,follow}`; `main_test.go` tests it. The scriptable verbs currently dispatch to `internal/client`. `dash`/`deploy`/`attach`/`ping` and the `raw` prefix are handled there too.
- Config/env: `config.Load(envMap(), ~/.rbg.conf)`; the store path for the new model is `~/.rbg/agents.json`; home is `os.UserHomeDir()`.

// Package cli is rbg's scriptable command layer: it parses non-interactive
// verbs, calls the Engine, and renders results. It depends on the Ops interface
// (satisfied by *engine.Engine) so it is unit-testable with a fake — no real
// SSH or claude. cmd/rbg builds a real Engine and routes verbs through Dispatch.
package cli

import (
	"fmt"
	"io"

	"github.com/divkov575/rbg/internal/core"
)

// Ops is the Engine surface the CLI drives. *engine.Engine satisfies it.
type Ops interface {
	List() ([]core.Agent, error)
	Create(spec core.Agent) (core.Agent, error)
	Run(name string) error
	Send(name, task string) error
	Read(name string) ([]byte, error)
	Kill(name string) error
	Adopt(name string) error
}

// Dispatch parses args (verb + operands), invokes ops, writes output to out,
// and returns a process exit code (0 = success, non-zero = error/usage).
func Dispatch(args []string, ops Ops, out io.Writer) int {
	if len(args) == 0 {
		fmt.Fprint(out, usage())
		return 2
	}
	verb, rest := args[0], args[1:]
	switch verb {
	default:
		fmt.Fprintf(out, "rbg: unknown command %q\n\n%s", verb, usage())
		return 2
	}
	_ = rest
}

// usage returns the scriptable-verb help text.
func usage() string {
	return `rbg — remote Claude agent management

Commands:
  ls                       list all agents (both machines)
  create <name> <repo> <task>   stage a held task
  run <name>               launch (or re-run) a staged agent, sync-first
  send <name> <task>       send a follow-up to a running agent
  read <name>              print an agent's transcript
  kill <name>              stop an agent (keeps transcript)
  adopt <name>             manage an agent started outside rbg
`
}
```

package cli

import (
	"fmt"
	"strings"

	"github.com/divkov575/rbg/internal/core"

// renderAgents formats the inventory as a compact, aligned table: one row per
// agent showing name, where it runs, its state, origin, and repo. Foreign
// agents are marked so the operator can tell them from managed ones.
func renderAgents(agents []core.Agent) string {
	if len(agents) == 0 {
		return "no agents\n"
	}
	var b strings.Builder
	fmt.Fprintf(&b, "%-20s  %-6s  %-8s  %-8s  %s\n", "NAME", "WHERE", "STATE", "ORIGIN", "REPO")
	for _, a := range agents {
		fmt.Fprintf(&b, "%-20s  %-6s  %-8s  %-8s  %s\n",
			a.Name, a.Where, a.State, a.Origin, a.Repo)
	}
	return b.String()
}
```

// doLs renders the reconciled inventory. A degraded list (unreachable machine)
// still renders the usable agents, prefixed with a warning, and exits non-zero.
func doLs(ops Ops, out io.Writer) int {
	agents, err := ops.List()
	if err != nil {
		fmt.Fprintf(out, "warning: inventory may be incomplete: %v\n", err)
	}
	fmt.Fprint(out, renderAgents(agents))
	if err != nil {
		return 1
	}
	return 0
}

func doCreate(rest []string, ops Ops, out io.Writer) int {
	if len(rest) != 3 {
		fmt.Fprintf(out, "usage: rbg create <name> <repo> <task>\n")
		return 2

		return 1

	return 0

// doName runs a one-name operation (run/adopt/kill), mapping a missing name to a
// usage error and an engine error to exit 1.
func doName(rest []string, out io.Writer, op func(string) error) int {
	if len(rest) != 1 {

	if err := op(rest[0]); err != nil {
		fmt.Fprintf(out, "rbg: %v\n", err)
		return 1
	}
	fmt.Fprintf(out, "ok: %s\n", rest[0])
	return 0

// doSend delivers a follow-up task from `send <name> <task>`. A busy agent is
// reported clearly (host.ErrBusy), distinct from other failures.
func doSend(rest []string, ops Ops, out io.Writer) int {
	if len(rest) != 2 {
		fmt.Fprintf(out, "usage: rbg send <name> <task>\n")
		return 2
	}
	if err := ops.Send(rest[0], rest[1]); err != nil {
		if errors.Is(err, host.ErrBusy) {
			fmt.Fprintf(out, "rbg: %q is busy — a send is already running\n", rest[0])
			return 3
		}
		fmt.Fprintf(out, "rbg: %v\n", err)
		return 1
	}
	fmt.Fprintf(out, "sent to %s\n", rest[0])
	return 0
}

// doRead prints an agent's transcript, rendering the raw JSONL to human text.
func doRead(rest []string, ops Ops, out io.Writer) int {
	if len(rest) != 1 {
		fmt.Fprintf(out, "usage: rbg read <name>\n")
		return 2
	}
	data, err := ops.Read(rest[0])
	if err != nil {
		fmt.Fprintf(out, "rbg: %v\n", err)
		return 1
	}
	render.Stream(strings.Split(string(data), "\n"), out)
	return 0
}
```

func main() {
	args := os.Args[1:]

	}

		args = args[1:]
		if len(args) == 0 {
			fmt.Fprintln(os.Stderr, "rbg: raw requires a verb")
			os.Exit(2)
		}
	}
	verb := args[0]
	// non-engine verbs keep their existing handlers.
	switch verb {
	case "deploy", "ping", "attach":
		os.Exit(runLegacy(verb, args[1:]))
	}
	if isScriptable(verb) {
		e, err := buildEngine()
		if err != nil {
			fmt.Fprintf(os.Stderr, "rbg: %v\n", err)
			os.Exit(2)
		}
		os.Exit(cli.Dispatch(args, e, os.Stdout))
	}
	fmt.Fprintf(os.Stderr, "rbg: unknown command %q\n", verb)
	os.Exit(2)
}

// isScriptable reports whether a verb is handled by the engine-backed CLI.
func isScriptable(verb string) bool {
	switch verb {
	case "ls", "create", "run", "send", "read", "kill", "adopt":
		return true
	}
	return false
}

// buildEngine loads config and constructs a real Engine over ssh/exec.
func buildEngine() (*engine.Engine, error) {
	cfg, err := config.Load(envMap(), os.ExpandEnv("$HOME/.rbg.conf"))
	if err != nil {
		return nil, err
	}
	if cfg.Mux {
		ensureControlDir(cfg.ControlPath)
	}
	home, _ := os.UserHomeDir()
	storePath := filepath.Join(home, ".rbg", "agents.json")
	return engine.New(cfg, run.Exec{}, storePath, home)
}
```

## Self-Review Notes (traceability to the HLD)

- **F9 (scriptable surface):** every engine op has a verb — `ls`/`create`/`run`/`send`/`read`/`kill`/`adopt` — through `cli.Dispatch`, which the binary routes to. ✅
- **F1/F2:** `create` (stage held) + `run` (launch, incl. a held record). ✅
- **F3:** `ls` (inventory), `adopt`. ✅
- **F4:** `send`, `kill`. ✅
- **F5:** `run` is sync-first (the engine pulls before launch); the CLI surfaces its error. ✅
- **F8:** `read` (rendered transcript). ✅
- **Testability (NFR):** `cli` depends on the `Ops` interface; all dispatch tests use `fakeOps` — no real SSH/claude. The `engine_assert.go` compile-time check keeps `Ops` honest against the real Engine. ✅
- **Thin front-end:** parsing/dispatch/render live in `internal/cli`; `cmd/rbg` only wires config + engine construction. ✅

**Design decisions (documented, deliberate):**
- `ls` on a degraded inventory still renders the usable agents and exits non-zero (mirrors the engine's usable-list-plus-error contract) — the operator sees what's reachable rather than nothing.
- `send` busy → exit 3 with a clear message (preserving the shipped rbg's busy exit-code convention), other errors → exit 1, usage errors → exit 2. Consistent exit-code scheme across verbs.
- The `raw` prefix is still accepted and stripped, so existing muscle memory / scripts (`rbg raw ls`) keep working.
- Positional args for `create <name> <repo> <task>` (not flags) match the shipped CLI's positional style; a repo-less held task (`create <name> "" <task>`) is possible but unusual — the engine allows empty Repo (sync is skipped on run).

**Explicitly deferred (Phase 4 / later, NOT gaps here):**
- The interactive **dashboard** rewrite onto the engine, which is what enables retiring `internal/client`, `internal/queue`, and `internal/tui`. Until then those packages and the `dash`/`deploy`/`attach`/`ping` verbs stay on their current implementations.
- A `run --new`/`CreateAndRun` one-shot convenience.
- Typed degradation errors so the UI can distinguish "remote unreachable" from other failures (noted in the engine review).
- Project/grouped `ls` views via `core.GroupByRepo` (dashboard territory).

**Type/name consistency:** `Ops`, `Dispatch`, `Usage`/`usage`, `doLs`/`doCreate`/`doName`/`doSend`/`doRead`, `renderAgents`, `isScriptable`, `buildEngine`, `runDash`/`runLegacy` — used identically across tasks; `Ops` matches `*engine.Engine`'s real method signatures (enforced by `engine_assert.go`). ✅
